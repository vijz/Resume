Problem

1. Why This Needed to Change

The practice maintains a certification database of 135+ members to track who holds which certifications, on which products, with what validity, and where the proof lives. Keeping it current was a recurring, high-effort manual task that included identifying who was missing details, drafting personalised requests, tracking replies, parsing freeform responses, updating the spreadsheet, and sending reminders without double-messaging people who had already responded.

Before automation, a single collection batch was taking 14 to 16 hours. Running three times a week, that was 42 to 48 hours of effort just on collection.

Key pain points before automation

Missed replies — people responded but their data never made it into the file. Inconsistent formats made parsing error-prone. No audit trail of who was contacted when. Reminders sometimes went to people who had already replied.

Solution

2. How the Workflow Works

A scheduled agentic workflow runs every Monday, Wednesday and Friday at 11:45am — a time chosen deliberately because I knew my laptop would be on. Each run performs the full collection cycle end-to-end:

  1. Load the pending set from the master Excel on SharePoint

  2. Classify each person by scanning Teams chat history from the last 14 days — categories are: Replied With Details, Replied None, Awaiting Reply, or New Contact

  3. Update tracker — for everyone who replied, certification details are written into the correct rows. Multi-cert respondents get extra rows inserted automatically

  4. Send reminders (Template B — gentle nudge) to people still awaiting reply

  5. Send first contacts (Template A — personalised table with their Emp Code, Name and Email pre-populated) to new people, capped at 30 per run

  6. Summarise the run to me in Teams — counts by category, exceptions flagged

  7. Log the run to a Run Summary sheet for full audit trail with timestamp

Design Decisions

3. What I Built In On Purpose

Safe-write design

The workflow always looks up Excel rows by email at write time — it never trusts cached row numbers. Updates happen in reverse-row order to avoid index drift when inserting rows for multi-cert respondents. The file saves after every batch of edits.

Safety guardrails

30 first-contacts maximum per run to prevent flooding. A skip list handles known-unresolvable emails. Reply check always happens before sending reminders, so no one gets chased twice. Every exception is logged separately for manual follow-up.

Reusable pattern

A reusable prompt template was built for this workflow so the same collection pattern can be applied to other practice databases — training tracker, project allocation, and similar recurring data-collection tasks — without rebuilding from scratch.
